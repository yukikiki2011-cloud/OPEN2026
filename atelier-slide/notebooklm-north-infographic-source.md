# NotebookLM ソース用 — 「NORTH／次の座標」インフォグラフィック設計文

この文書は NotebookLM のインフォグラフィック生成の**入力ソース**として使う。図解では「宗教・神秘・星・光線・モノリス」などの象徴は使わない。**ビジネス・組織・プロダクト**の語彙で整理すること。

---

## 1. ひと言で言うと何か

**雪舟館 Anima**という AI 組織が、Git リポジトリ `sesshuu-anima` を「組織そのものの設計書」として運用し、その上で **Claude Code を核とした制作ライン**と **コミュニティ・外向き発信**を同時に進める。「次にどこへ進むか」を **NORTH（北極星メタファーではなく、方角・優先順位の比喩として）** と呼んでいる。

---

## 2. 主体と設立

- **社名**: 雪舟館 Anima  
- **設立**: 2026年  
- **代表**: sesshuu design  
- **理念（原文の意図）**: AI は人間の**代替**ではなく**能力拡張**。人間の美意識と戦略眼を、AI でスケールさせる。  
- **事業の4本柱**: **AI × デザイン × SEO × コンサル** — クライアント向けサービスも、自組織の運用も、この4つが交差する場として設計されている。

---

## 3. `sesshuu-anima` リポジトリの役割（最重要）

この Git リポジトリは「コード置き場」ではなく、**組織設計書そのもの**として扱う。

- ルートの **`CLAUDE.md`** は AI 側の**最高規範（COO層）**: 目的から逆算し、構造を整え、成果に繋げる。  
- **`SNOW/`** は議事・STATUS・スケジュール — 文脈の接続役。  
- **TOMOYA／TERAMACHI／KAWAI／YANAGI／KIUCHI** 各フォルダに **役割ごとの憲法・SKILLS** がある。  
- **`presentations/`** に HTML デッキやポスターがあり、**外向きの説明**と**内向きの記録**を同じリポで更新する。

**キーメッセージ（図解向け）**:  
「**One repository = one source of truth for the organization**」— 憲法・スキル・議事・プレゼンをバラバラのファイル共有にしない。

---

## 4. 二本のエンジン — TOMOYA と KIUCHI

組織図上、役割は分かれているが、**進路は二本線で同じ地平を目指す**イメージ。

### TOMOYA（戦略参謀・Claude Code 側）

- **内訳**: ブランド戦略、情報構造、KPI、提案書の骨格、検索意図、クライアント向け論理の組み立て。  
- **ツール**: Claude Code。  
- **この文脈での意味**: 「**構造**」— 何を作るか・どう測るか・どう並べるか。**本番サイト**（例: `sesshuu-portfolio`）と**試作**（例: `sesshuu-portfolio-logomotion`）の分離も、再現可能な型として扱う。

### KIUCHI（経営戦略・コミュニティ）

- **内訳**: SHIFT AI コミュニティ圏との関係、マーケ・教育動線、対外ストーリー。  
- **この文脈での意味**: 「**場**」— Discord / X / イベント / 認定施策など、**外に届く接続**。**SHIFTpro**（投稿チャレンジや週次の小さな儀式）と雪舟館の活動を接続する。

**図解の推奨比喩**:  
左右二列の「**構造（TOMOYA）**」と「**場（KIUCHI）**」、中央または下部で「**sesshuu-anima リポジトリ**」が両方を束ねる。**宗教画ではなく、組織図または二系統パイプライン**。

---

## 5. ワンライナー・キーワード（ビッグワード）

プレゼン最終スライドやポスターで使っている短語。インフォでも**階層（大→小）**を意識する。

| 優先度 | 文言 | 意味 |
|--------|------|------|
| 最大 | **能力拡張** | 理念の核（代替ではなく拡張） |
| 大 | **TOMOYA × KIUCHI** | 二本線の象徴的表記 |
| 中 | **one repo · one truth** | 単一リポジトリの原則 |
| 中 | **戦略 ⊗ 場** | 内と外の同時設計 (⊗ は掛け合わせのイメージ) |
| 小 | **4 axes** | AI・デザイン・SEO・コンサル |
| 小 | **SHIFTpro 接続** | コミュニティ側の受け口 |
| 小 | **静謐 · 構造 · 美** | ブランド DNA（饒舌にならないトーン） |

## 6. 外向きの共有 URL（運用）

- デッキ HTML を **`sesshuu-anima`** にコミットし、**GitHub Pages** の公開ルートを **`presentations/sesshuu-claude-journey/`** に向ける。  
- その URL が**そのまま共有リンク**になる。画像は `images/` フォルダに同梱し、コミット漏れしないようにする。  
- リポジトリが private の場合は、組織の `SNOW/STATUS.md` の方針に沿って公開方法を決める（ミラー等）。
